# SolidJS

Use this reference for SolidJS, SolidStart, signals, stores, resources, and fine-grained reactive UI.

Use signals and stores in the idiomatic Solid style. Avoid destructuring reactive props in ways that lose tracking. Use `createMemo`, `createResource`, `Show`, `For`, and error boundaries when they improve clarity. Keep effects for synchronization, not derived state.

Follow SolidStart routing and server function conventions when present. Keep browser-only behavior guarded during SSR and preserve accessibility in custom controls.
