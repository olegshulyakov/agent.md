# Astro

Use this reference for Astro sites and apps using content collections, islands, static generation, server rendering, and framework integrations.

Prefer static HTML and Astro components by default. Add client islands only for interaction that requires browser JavaScript, and choose the narrowest hydration directive that works. Keep content schemas, frontmatter, image handling, and routing aligned with the repository.

Use framework components inside islands according to the integration already present. Preserve accessibility and performance by avoiding unnecessary client bundles and by using Astro image/content utilities where available.
