# Svelte

Use this reference for Svelte components, stores, actions, transitions, and Vite Svelte apps.

Keep reactive declarations small and predictable. Use stores for shared state and component state for local interaction. Avoid hidden mutations that make reactive dependencies unclear. Keep actions focused on DOM integration and clean up event listeners or observers.

Follow existing component, slot, event, and styling conventions. Use transitions sparingly and respect reduced-motion preferences. Preserve semantic markup before reaching for custom interactive elements.
