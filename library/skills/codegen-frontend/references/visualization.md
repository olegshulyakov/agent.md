# Visualization

Use this reference for charts, dashboards, dense tables, maps, timelines, KPI panels, and interactive data exploration.

Use the charting or table library already present. Choose chart types that match the comparison: lines for time, bars for categories, stacked charts only when composition matters, scatterplots for relationships, and tables for exact values. Avoid decorative charts that obscure the task.

Make states explicit for loading, empty, partial data, stale data, errors, and filtered results. Keep legends, axes, units, timezone, aggregation, and sorting visible. Provide accessible names, summaries, keyboard access where supported, and a tabular fallback or export when the app convention supports it.
