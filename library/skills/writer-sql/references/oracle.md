# Oracle Reference

## Key types

- `NUMBER(p,s)` — exact numeric (both integers and decimals)
- `VARCHAR2(n CHAR)` — variable-length string (prefer CHAR semantics over BYTE)
- `NVARCHAR2(n)` — Unicode strings
- `DATE` — stores date and time (not just date!)
- `TIMESTAMP WITH TIME ZONE` — timezone-aware timestamps (prefer over DATE)
- `CLOB` / `NCLOB` — large text
- `BLOB` — binary data
- `RAW(16)` — UUIDs (or use `VARCHAR2(36)` with string format)

## Sequences and identity

```sql
-- Traditional sequence
CREATE SEQUENCE user_seq START WITH 1 INCREMENT BY 1 NOCACHE NOCYCLE;

CREATE TABLE users (
    id NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    email VARCHAR2(255 CHAR) NOT NULL,
    CONSTRAINT users_email_uq UNIQUE (email)
);

-- UUID equivalent
CREATE TABLE users (
    id RAW(16) DEFAULT SYS_GUID() PRIMARY KEY,
    ...
);
```

## PL/SQL procedures

```sql
CREATE OR REPLACE PROCEDURE create_user(
    p_email  IN VARCHAR2,
    p_name   IN VARCHAR2,
    p_id     OUT RAW
) AS
BEGIN
    p_id := SYS_GUID();
    INSERT INTO users (id, email, name, created_at)
    VALUES (p_id, p_email, p_name, SYSTIMESTAMP);
    COMMIT;
EXCEPTION
    WHEN DUP_VAL_ON_INDEX THEN
        RAISE_APPLICATION_ERROR(-20001, 'Email already exists');
    WHEN OTHERS THEN
        ROLLBACK;
        RAISE;
END create_user;
/
```

## Dual table

Used for expressions without a real table:

```sql
SELECT SYSDATE FROM DUAL;
SELECT SYS_GUID() FROM DUAL;
SELECT UPPER('hello') FROM DUAL;
```

## Upsert (MERGE)

```sql
MERGE INTO users tgt
USING (SELECT :email AS email, :name AS name FROM DUAL) src
ON (tgt.email = src.email)
WHEN MATCHED THEN
    UPDATE SET tgt.name = src.name, tgt.updated_at = SYSTIMESTAMP
WHEN NOT MATCHED THEN
    INSERT (id, email, name, created_at, updated_at)
    VALUES (SYS_GUID(), src.email, src.name, SYSTIMESTAMP, SYSTIMESTAMP);
```

## Pagination (12c+)

```sql
-- Row limiting clause (Oracle 12c+)
SELECT id, email, created_at
FROM users
ORDER BY created_at DESC
OFFSET :offset ROWS FETCH NEXT :limit ROWS ONLY;

-- ROWNUM approach (older)
SELECT * FROM (
    SELECT a.*, ROWNUM rn FROM (
        SELECT id, email FROM users ORDER BY created_at DESC
    ) a WHERE ROWNUM <= :end_row
) WHERE rn > :start_row;
```

## CTEs

```sql
WITH active_users AS (
    SELECT id, email FROM users WHERE deleted_at IS NULL
),
order_stats AS (
    SELECT user_id, COUNT(*) AS cnt FROM orders GROUP BY user_id
)
SELECT u.email, NVL(o.cnt, 0) AS order_count
FROM active_users u
LEFT JOIN order_stats o ON o.user_id = u.id;
```

## Hints

Use hints sparingly — prefer fixing statistics or indexes first:

```sql
SELECT /*+ INDEX(u idx_users_email) */ *
FROM users u WHERE email = :email;

SELECT /*+ PARALLEL(o, 4) */ * FROM orders o;
```

## EXPLAIN PLAN

```sql
EXPLAIN PLAN FOR
SELECT * FROM orders WHERE user_id = :uid;

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

## Partitioning

```sql
CREATE TABLE orders (
    id RAW(16) DEFAULT SYS_GUID(),
    created_at DATE NOT NULL,
    total NUMBER(15,4)
)
PARTITION BY RANGE (created_at)
INTERVAL (NUMTOYMINTERVAL(1, 'MONTH'))
(PARTITION p_initial VALUES LESS THAN (DATE '2024-01-01'));
```

## Common gotchas

- Empty string `''` equals NULL in Oracle — there is no distinction
- `VARCHAR2` max is 32767 bytes in PL/SQL, 4000 bytes in SQL (use CLOB beyond that)
- Date arithmetic: `SYSDATE + 1` adds 1 day; `SYSDATE + 1/24` adds 1 hour
- NVL vs COALESCE: both work; COALESCE is ANSI-standard
- String concatenation: use `||` operator
